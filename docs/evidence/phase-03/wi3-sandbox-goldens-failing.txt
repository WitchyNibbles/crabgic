Setup (deliberately-wrong stub, per this worker's TDD protocol — "prefer
genuine assertion-level failures over module-absence failures"):
  - `src/compiler/sandbox-profile.ts` shipped with `emitSandboxProfile`
    ignoring its `envelope` argument entirely and returning
    `network.allowAllUnixSockets: false` (should always be `true` on
    Linux/WSL2 per docs/engine-baseline.md §6) and an empty
    `filesystem.denyRead` (drops every mandatory deny path). The
    `SandboxProfileSchema.parse(...)` call inside the stub itself throws a
    ZodError on `allowAllUnixSockets: false` (its schema requires the
    literal `true`) — every downstream consumer (`compile-envelope.ts`,
    `worker-settings.ts`, `generate-golden-artifacts.ts`) that calls
    `emitSandboxProfile` therefore also fails, cascading a single seeded
    defect across every wi3-scoped test file.
  - The golden files under `packages/engine-core/goldens/` do not exist
    yet at all (`ls packages/engine-core/goldens/` — empty directory) —
    the roadmap's own sanctioned failing-first form for this sub-part
    ("Failing-first: a byte-diff test against a golden file that doesn't
    exist yet"). In this run that ENOENT never gets reached because the
    ZodError above throws first inside `buildGoldenArtifacts()` — a
    stronger RED than a missing-file read, not a weaker one.
