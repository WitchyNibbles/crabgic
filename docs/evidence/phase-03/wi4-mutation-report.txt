Method: each seeded variant is a function, exported ONLY from
`mutation.test.ts` (never from `src/` production code), that takes the
REAL `compileEnvelope`'s own correct output (a `CompiledWorkerProfile`
built from a non-trivial fixture envelope — owned paths + two authorized
Bash commands) and mutates it to reintroduce exactly one seeded defect.
Each seed is verified twice: (a) the real compiler's own unmutated output
never trips the corresponding invariant assertion (a negative control —
proves the assertion isn't simply always-throwing), and (b) the mutated
variant IS caught by that same assertion, throwing the expected typed
error.
