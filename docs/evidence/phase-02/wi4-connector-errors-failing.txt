Note on the type-level security assertion specifically: the `@ts-expect-error`
lines in connector-error.test.ts assert against a type (`ConnectorErrorData`)
and class (`ConnectorError`) that, at this failing-first stage, do not exist
at all — the module fails to resolve before type-checking is even reached,
which is itself the strongest possible "failing" state for those assertions.
Once the implementation lands, the type-level guarantee is verified
separately via `npx tsc -b packages/contracts` (vitest transpiles with
esbuild and does not enforce `@ts-expect-error`) — see
wi4-connector-errors-passing.txt for that run.
